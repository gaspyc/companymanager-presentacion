Estructura del Combo

Cada regla
define qué elegir.

Un combo puede tener una o varias reglas. Cada regla tiene un target_type que determina si el cliente elige de una categoría entera o si el producto ya viene fijo. Una promo puede mezclar ambos tipos.

El campo quantity de cada regla define cuántas unidades hay que completar para que esa regla quede satisfecha.

Producto fijo (target_type: sku)

El ítem ya está definido. No hay nada que elegir. El modal pre-selecciona la cantidad automáticamente.

{ target_type: "sku", target_value: "INV-001", quantity: 2 }
Categoría abierta (target_type: category)

El cliente elige libremente entre todos los productos de esa categoría hasta completar la cantidad indicada.

{ target_type: "category", target_value: "Aromatizantes", quantity: 3 }
Combo mixto

El array raw_items puede combinar reglas fijas y de categoría. El modal las muestra como secciones separadas.

raw_items: [ /* regla fija */, /* regla cat. */ ]
El Sistema de Selección

No podés
pasarte ni
quedarte corto.

Cada regla tiene su propio contador. El botón + se bloquea en cuanto la regla se completa — no se puede agregar más de lo que la promo permite. Y el botón de confirmar solo se habilita cuando todas las reglas están completas.

La validación corre en el computed isComplete que recorre cada regla y chequea las sumas.

Fijo
Regla incompleta → botón bloqueado

Mientras alguna regla no alcance su cantidad mínima, el botón de confirmar queda gris y con cursor: not-allowed. El texto dice "Completá las opciones para continuar".

Regla completa → + bloqueado

Cuando el total seleccionado para una regla llega a su límite, updateSelection hace return sin hacer nada. No hay alert, no hay error.

Todas completas → botón verde

isComplete es true. El botón cambia a verde y emite @confirm con el combo, la cantidad y la configuración elegida.

Buscador en tiempo real

El campo de búsqueda filtra los productos elegibles dentro de cada regla. No afecta las selecciones ya hechas — solo oculta opciones visualmente.

Dos caminos posibles

El modal se abre
solo cuando hace falta.

Autoconfirm (combo fijo)

Si todas las reglas son de tipo sku, no hay nada que elegir. El catálogo genera la configuración automáticamente y llama a addToCartWithConfig directo, sin abrir el modal.

1
Cliente toca "Agregar"
2
Catálogo detecta que es combo fijo
3
Genera config automática
4
Agrega al carrito directo + toast
Modal manual (combo abierto)

Si alguna regla es de tipo category, el cliente tiene que elegir. El catálogo abre el ComboBuilderModal y espera el evento @confirm.

1
Cliente toca "Agregar"
2
Catálogo detecta que tiene categorías
3
Abre ComboBuilderModal
4
Cliente elige → @confirm → carrito
La lógica que detecta si el combo es fijo o abierto vive en StoreCatalog, en la función handleAddProductClick. El modal en sí no sabe nada de esto — solo recibe el combo y espera confirmación.
Interfaz del Componente

Props, eventos y lo que emite.

PropTipoDescripción
show Boolean Controla si el modal está visible. El watch sobre este prop limpia las selecciones al abrirse. default: false
combo Object El objeto combo con name, description, price y raw_items (las reglas). default: null
products Array Catálogo completo de la tienda. Se usa para filtrar los productos elegibles por categoría o SKU. Required
initialQuantity Number Cuántos combos quiere agregar el cliente. Se pasa directo al evento @confirm. default: 1
hidePrices Boolean Oculta el badge de precio del combo en el hero del modal. default: false
Payload del evento @confirm
{
  combo: { name: "Combo Verano x3", price: 14900, ... },
  quantity: 1,
  configuration: [
    { product_sku: "INV-001", quantity: 1 },
    { product_sku: "VAI-003", quantity: 2 }
  ]
}
