Carga Instantánea

Sin spinner
cuando viene
del catálogo.

Cuando el cliente toca una card en StoreCatalog, esta página no parte de cero. Lee un caché en memoria con readStorefrontPreview(storeRef, sku) que ya tiene el perfil, el producto y el catálogo completo guardados.

saveStorefrontPreview() en el origen

StoreCatalog guarda perfil + producto + catálogo antes de navegar, junto con sourceScrollY para poder volver al mismo punto de scroll.

hydrateFromPreview() síncrono

Se ejecuta antes del primer render. Si hay caché válido, loading = false desde el inicio — no hay flash de spinner.

Fetch en segundo plano

Aunque hidrate desde caché, igual dispara api.getPublicStorefront() para refrescar datos por si cambiaron desde que se guardó el preview.

Acceso directo por URL

Si no hay caché (link compartido, refresh, Google), carga normal con loading visible y fetch completo antes de mostrar contenido.

SEO Estructurado

Un solo
JSON-LD con
cuatro nodos.

El @graph conecta breadcrumbs, la página, el local y el producto en un solo bloque de schema.org. Google puede mostrar rich snippets con precio, disponibilidad y la cadena de navegación completa.

BreadcrumbList

CompanyMarket → Nombre de la tienda → Nombre del producto. Tres niveles con URLs absolutas.

WebPage

Conecta con isPartOf al WebSite raíz y referencia el breadcrumb por @id.

LocalBusiness

Nombre, teléfono e imágenes del local. Mismo @id que usaría el perfil — Google los relaciona como la misma entidad.

Product + Offer

Si hidePrices está activo, el nodo offers directamente no se incluye — no hay precio falso ni oculto en el schema.

{
  "@context": "https://schema.org",
  "@graph": [
    { "@type": "BreadcrumbList" ... },
    { "@type": "WebPage" ... },
    { "@type": "LocalBusiness" ... },
    {
      "@type": "Product",
      "name": "Invictus · Aura Lotion",
      "sku": "INV-AURA-250",
      "offers": {
        "@type": "Offer",
        "priceCurrency": "ARS",
        "price": "7475.00",
        "availability": "https://schema.org/InStock"

Reutilización Total

Debajo del hero,
la tienda
completa otra vez.

StoreProduct no reinventa nada para mostrar "más productos de esta tienda". Importa StoreProfile y StoreCatalog directamente — los mismos componentes que arma la vidriera pública — con una ref al catálogo para controlarlo desde el hero.

Cero duplicación de UI

El perfil y el catálogo se renderizan tal cual aparecen en la tienda principal. Cualquier cambio visual en esos componentes se propaga automáticamente.

catalogRef.addProductBySku

Cuando el cliente agrega desde el hero, StoreProduct llama al método expuesto del catálogo embebido para mantener un solo estado de carrito compartido.

focusProductBySku

Resalta visualmente la card del producto actual dentro del catálogo de abajo, para que el cliente vea de dónde viene en el contexto completo.

heroCardRef + getStickyOffset

Calcula la altura del navbar público para ajustar el scroll suavemente al hero, sin que quede tapado al volver desde una navegación con offset.
