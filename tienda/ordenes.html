La Feature Estrella

PIN + QR
para confirmar
la entrega.

Cuando un pedido de delivery está en camino, el cliente puede ver un PIN de 4 dígitos y un QR generado en el cliente. El repartidor los escanea o ingresa el PIN para confirmar la entrega sin llamadas.

1
Pedido de delivery en tránsito

El orden tiene shipping_method = "delivery" y delivery_pin asignado. Aparece una banda verde en la card.

2
Cliente toca la card

handleOrderClick detecta la condición y abre el modal. Genera el QR en el cliente con el payload de entrega.

3
Polling cada 4 segundos

Mientras el modal está abierto, consulta el historial silenciosamente. Si el pedido pasa a completed o delivered, el modal se cierra solo.

4
Entrega confirmada

El polling detecta el cambio de estado. El modal desaparece y la card del pedido se actualiza a "Completado" sin recargar la página.

Carritos Guardados

El carrito
no se pierde
nunca.

Cada vez que el cliente agrega un ítem en cualquier tienda, el carrito se guarda en localStorage con una key por tienda. En "Mis Pedidos", el componente los recupera todos y busca el perfil de cada tienda para mostrar nombre y logo.

¿Cómo funciona la recuperación?
Al montar, se listan todas las keys de localStorage que empiecen con market_cart_. Por cada una se hace un getPublicProfile(businessId) en paralelo con Promise.allSettled para no fallar si un local fue dado de baja.

Tracking en Vivo

GPS del repartidor
directo en la pantalla.

Mientras hay un delivery activo, el componente refresca las coordenadas del repartidor cada 20 segundos. El mapa SVG renderizado en el cliente muestra tres puntos: local, repartidor y destino.

Señal en vivo (≤5 min)

El punto del repartidor aparece verde con un halo animado y el label dice "Última señal hace X min".

Señal vencida (>5 min)

El punto cambia a amarillo y el label advierte "Señal sin actualizar hace X min". No desaparece — muestra la última posición conocida.

Sin coordenadas

Si el backend no envió ubicación todavía, el mapa muestra solo local y destino con el overlay "Esperando señal del repartidor".

Intervalos y Tema

Dos loops
independientes.
Un tema heredado.

El componente maneja dos ciclos de polling simultáneos con intervalos distintos, y se limpia solo al desmontarse. Además acepta un prop theme del storefront premium que repinta toda la UI.

PIN polling — cada 4 seg

Solo activo mientras el modal de PIN está abierto. Si el pedido se completa, cierra el modal automáticamente.

pinPollingInterval
Tracking polling — cada 20 seg

Refresca el historial silenciosamente y actualiza las coordenadas del repartidor. El reloj interno se actualiza cada minuto para el label de antigüedad.

trackingRefreshInterval
onUnmounted — limpieza total

Ambos intervalos se limpian al desmontar. No hay memory leaks aunque el usuario cierre el panel a mitad de entrega.
