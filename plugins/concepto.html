El Cambio de Paradigma

El Problema que Resuelve

Antes, cada integración nueva implicaba meses de desarrollo propio, riesgo técnico y sin garantía de que sirva para todos los negocios.

Software Cerrado
Cada integración requiere modificar el core
Un bug en una integración rompe todo el sistema
Solo el equipo interno puede agregar funciones
Tiempo de entrega: meses por feature
El costo de desarrollo lo absorbe la plataforma sola
Ecosistema de Plugins
Cada plugin vive aislado; el core nunca se toca
Un plugin roto no afecta a los demás ni al sistema
Cualquier desarrollador puede publicar y monetizar
Tiempo de entrega: días desde que el dev termina
Los devs absorben su propio costo y comparten ganancia

Diseño Técnico

Arquitectura en Capas

El sistema está diseñado en capas que se comunican hacia arriba, nunca hacia abajo. Un plugin nunca puede tocar la base de datos directamente. Todo pasa por la API oficial.

Esto garantiza seguridad, auditabilidad y estabilidad para todos los comercios por igual.

El Ecosistema en Acción

¿Qué puede resolver un Plugin?

Cada plugin ocupa un rol específico dentro del sistema. Una sede puede tener un plugin distinto para cada rol.

Facturación AFIP, factura electrónica, comprobantes A/B/C. Se activa por sede.
Pagos MercadoPago, Stripe, transferencias. El plugin maneja el gateway externo.
Precios Dinámicos Reglas de pricing automáticas, descuentos por volumen, listas de mayorista.
Bot / Broadcast Automatización de WhatsApp, notificaciones, seguimiento de pedidos.
Extractor / Importador Lee archivos de proveedores (Excel, CSV, PDF) y los traduce al formato interno.
Logística Integración con correos, cálculo de envío, seguimiento de despachos.

De Punta a Punta

El Ciclo de Vida Completo

Un plugin nace como ZIP, pasa por auditoría, se publica en el Marketplace y finalmente se activa en la sede del comercio que lo necesita.

Cada paso tiene controles independientes: el dev decide el precio, el admin decide si aprueba, el comercio decide en qué sede activa.
