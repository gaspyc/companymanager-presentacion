La Arquitectura

Cómo Fluye la Comunicación

El core emite eventos al bus. Tu plugin los escucha, consulta la API para obtener datos y devuelve una respuesta. Nunca hay contacto directo entre tu código y la base de datos.

El bus actúa como intermediario: desacopla al plugin del sistema y permite que múltiples plugins reaccionen al mismo evento.

La Caja de Herramientas

Métodos de la API Oficial

La API expone funciones tipadas que reemplazan el acceso directo a la BD. Cada método valida que el permiso esté declarado en el manifest.json antes de ejecutarse.

Si el permiso no está declarado, el método lanza una excepción que el sistema registra para auditoría.

GET
api.getProducts(filters?) Devuelve el catálogo. Requiere read:products. Soporta filtros por categoría, proveedor y estado.
GET
api.getOrder(orderId) Trae una orden completa con ítems, cliente y pagos. Requiere read:orders.
POST
api.emitInvoice(payload) Registra un comprobante fiscal generado. Requiere emit:invoice. Dispara on_invoice_emit.
HOOK
api.resolvePrice(ctx) → price Intercepta el cálculo de precio antes del POS. Devuelve el precio final. Requiere hook:price.
POST
api.notify(channel, message) Envía una notificación interna al comercio (banner, toast). Requiere emit:notify.
GET
api.getConfig(key) Lee la configuración propia del plugin guardada en su espacio aislado. Sin permisos extra.

Caso Real

Plugin de Facturación:
De Principio a Fin

Así funciona FacturAR Pro desde que el cajero cierra una venta hasta que el comprobante queda registrado en AFIP.

Cinco pasos, cero acceso directo a la base de datos, trazabilidad completa.

1
El cajero cierra la venta

El core termina la transacción y emite on_sale_complete al bus con el payload completo.

2
El plugin recibe el evento

FacturAR Pro estaba suscripto. Recibe el payload: items, total, cliente y medio de pago.

3
Consulta datos faltantes

Llama a api.getOrder(id) para obtener el CUIT del cliente y el tipo de comprobante requerido.

4
Llama a AFIP externamente

Con los datos completos, hace un POST al web service de AFIP y recibe el CAE del comprobante.

5
Registra el resultado

Llama a api.emitInvoice(payload) con el CAE. El sistema guarda el comprobante y dispara on_invoice_emit.
